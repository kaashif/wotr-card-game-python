# Repo Instructions

- Do substantial work in a separate git worktree on a dedicated branch.
- Commit and push completed chunks eagerly so progress is preserved remotely.
- Open a draft GitHub pull request for work in progress so it can be tracked.
- Before merging a work branch into the repo's primary branch, ask the user for confirmation.
- This repo currently uses `master` as its primary branch; treat user references to `main` as the primary branch unless the repo is renamed.
